Pages app¶

Introduction¶

Pages and modules are the main usable substance of our webapp. Everything related to them is handled in this app. Everything from creating through editing to deleting them. And of course also the different ways pages can be viewed.

class Page(*args, **kwargs)¶

Bases: django.db.models.base.Model

A Page is a collection of Modules with some extras

A Page has a title and is linked to the django.contrib.auth.User who created it through the user field. It knows whether it is active, visible, an example. It can have a token for sharing through a secret url.

It can contain any number of any type of modules, and has the ability to sort them by their position.

Module types:

  • pages.models.GeneralInfoModule
  • pages.models.CommunicationModule
  • pages.models.DoDontModule
  • pages.models.MedicationModule
  • pages.models.SensoryModule
  • pages.models.ContactModule
  • pages.models.FreeTextModule
  • pages.models.FreeListModule
  • pages.models.FreePictureModule

class CommunicationMethods(*args, **kwargs)¶

Bases: django.db.models.base.Model

CommunicationMethods belong to a pages.models.CommunicationModule. They describe prefered communication methods in two directions (‘me to you’ and ‘you to me’) for a certain situation.

The fields you_to_me_choices and me_to_you_choices both use METHODS as choices.

class MedicationItem(*args, **kwargs)¶

Bases: django.db.models.base.Model

MedicationItems belong to a pages.models.MedicationModule.

Each MedicationItem has a name, and a field for remarks

They can contain multiple pages.models.MedicationIntake, with further information.

They have a fixed position within the pages.models.MedicationModule, depending on the order they were added. This position is determined by the position field which is set automatically in the save method.

save(*args, **kwargs)¶

If creating

sets position field to the current maximum of positions within the pages.models:MedicationModule +1 and saves the object.

Else just save the object

templatetags pages_tags.py¶

get_display_value(choices, request_key)¶

Returns the display value of an item of a ChoiceArrayField.

This is because the standard django way <field_name>_display doesn’t work in this case. Needs the choices tuple and the key one wants the display_value of.

example usage:

{% for item in module.field_name %}
    {{ module.CHOICES|get_display_value:item }}
{% endfor %}
verbose_name(obj)¶

Returns the verbose name of the model this object is an instance of.

sensitivity_desc(value, sense)¶

Returns a verbose and localised sensitivity description

Needs a value that is part of pages.models.SensoryModule RANGE and a string sense: (sound, light, smell or temperature)

example usage:

{{ module.light|sensitivity_desc:"light" }}
sensitivity_img(value, sense)¶

Returns the svg image belonging to the sensitivity value and sense.

Needs a value that is part of pages.models.SensoryModule RANGE and a string sense: (sound, light, smell or temperature)

example usage:
